# BUSA6221 · LU5 Study Portal

> A dark-themed, interactive study reference for **Business Analysis Core** — Module 2A.  
> Covers four key frameworks: Stakeholder Layers, Mendelow's Matrix, RASCI, and CATWOE.

---

## 📖 About This Project

This is a single-file HTML study portal built for the **BUSA6221** module at university level. It requires no frameworks, no build tools, and no server — just open the file in a browser and start studying.

### Frameworks Covered

| Tab | Framework | Purpose |
|-----|-----------|---------|
| 01 | **Stakeholder Layers** | Internal, External, and Regulatory classification |
| 02 | **Mendelow's Matrix** | Power/Interest grid for stakeholder communication strategy |
| 03 | **RASCI** | Accountability and ownership model for project tasks |
| 04 | **CATWOE** | Worldview analysis for resolving conflicting requirements |

## 🚀 Deploying to the Internet with GitHub Pages

GitHub Pages is a **free** hosting service by GitHub. It turns any repository into a live website. Follow these steps exactly.

---

### Step 1 — Create a GitHub Account

1. Go to **[github.com](https://github.com)**
2. Click **"Sign up"** in the top-right corner
3. Enter your email, create a password, and choose a username
4. Verify your email address when prompted
5. You now have a free GitHub account ✅

> Skip this step if you already have an account — just sign in.

---

### Step 2 — Create a New Repository

A **repository** (or "repo") is like a folder on GitHub that stores your project files.

1. Once signed in, click the **"+"** icon in the top-right corner
2. Select **"New repository"**
3. Fill in the details:
   - **Repository name:** `busa6221-study-portal` (or any name you like — no spaces)
   - **Description:** `Interactive study portal for BUSA6221 LU5` (optional)
   - **Visibility:** Select **Public** ← *required for free GitHub Pages*
   - Leave everything else as-is
4. Click **"Create repository"** ✅

---

### Step 3 — Upload Your Files

You'll now upload your two files: `index.html` and `README.md`.

1. On your new repository page, click **"uploading an existing file"** (shown in the middle of the page)
   - Or click **"Add file" → "Upload files"** from the top menu
2. Drag and drop both files into the upload area:
   - `index.html` ← the renamed study portal
   - `README.md` ← this file
3. Scroll down to the **"Commit changes"** section
4. Leave the default commit message or type something like `Initial upload`
5. Click **"Commit changes"** ✅

Your files are now saved on GitHub.

---

### Step 4 — Enable GitHub Pages

This step turns your repository into a live website.

1. On your repository page, click **"Settings"** (tab near the top, with a gear icon)
2. In the left sidebar, scroll down and click **"Pages"**
3. Under **"Branch"**, click the dropdown that says `None`
4. Select **`main`** from the list
5. Leave the folder set to **`/ (root)`**
6. Click **"Save"** ✅

GitHub will now build your site. This takes **1–3 minutes**.

---

### Step 5 — Get Your Live URL

1. Stay on the **Pages** settings page (or navigate back to it)
2. After a minute or two, refresh the page
3. You will see a green banner at the top saying:

   > **"Your site is live at https://YOUR-USERNAME.github.io/busa6221-study-portal/"**

4. Click that link — your study portal is now **live on the internet** 🎉

> Your URL will follow this format:  
> `https://YOUR-GITHUB-USERNAME.github.io/REPOSITORY-NAME/`

---

### Step 6 — Share It

Copy the URL from Step 5 and share it with anyone — classmates, lecturers, study groups. No login required to view it.

## ✏️ Making Changes Later

If you want to update your portal after it's live:

1. Go to your repository on GitHub
2. Click on `index.html` in the file list
3. Click the **pencil icon** (Edit) in the top-right of the file view
4. Make your changes directly in the browser editor
5. Scroll down and click **"Commit changes"**
6. Your live site will automatically update within **1–2 minutes** ✅
